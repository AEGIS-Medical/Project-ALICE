Veritas AI — Data Flow Diagrams

Three views: File upload processing, video call platform integrations, and end-to-end user experience

Manual File Upload — Processing Pipeline
User uploads a pre-recorded video file (MP4, MOV, WebM, MKV). The system ingests, validates, compresses, then runs the full tri-vector analysis asynchronously. Results are delivered as a timestamped report.
Mobile Client
📱
File Picker
Android: SAF / iOS: PHPicker
User selects video file from device storage, camera roll, or cloud storage (Google Drive, iCloud, Dropbox).
✅
Client Validation
Ktor + MediaCodec probe
File type check (MP4/MOV/WebM/MKV). Size cap enforcement (2GB max). Duration check (≤4 hours). Corrupt file detection via header probe.
🗜️
On-Device Compress
H.265/HEVC via MediaCodec
Re-encode to 720p@24fps CRF 28 if source exceeds quality threshold. Skip if already ≤720p. Reduces upload size 40-60%.
Optional
TLS 1.3
📤
Chunked Upload
TUS Protocol / Resumable
Resumable chunked upload (5MB chunks). Progress bar on client. Auto-retry on network interruption. SHA-256 checksum per chunk.
Ingestion Layer
🚪
API Gateway
FastAPI — POST /uploads/video
JWT auth validation. Rate limit check (5 uploads/hour). Chunk reassembly. Full-file integrity verification. Assigns job_id.
📦
Object Store Write
MinIO — uploads/{user}/{job_id}/
Original file stored in hot tier. AES-256-GCM encryption at rest with per-user key from Vault. Metadata written to PostgreSQL.
📨
Job Queue Dispatch
Celery via Redis
Analysis job queued with priority (paid tier = higher). WebSocket channel opened for client to receive progress updates.
Extraction Pipeline
🎬
FFmpeg Demux
ffmpeg -i input.mp4
Separates video and audio streams. Extracts duration, framerate, codec info. Produces PCM audio (16kHz mono) for transcription.
PARALLEL EXTRACTION
👁️
Video → Landmarks
MediaPipe Face Mesh (server)
Extract 478 facial landmarks per frame at native fps. Output: protobuf landmark stream with frame timestamps.
🎤
Audio → Transcript
faster-whisper large-v3
Demucs source separation first (isolate voice from bg). Then Whisper with word-level timestamps. Language auto-detect.
🔊
Audio → Embeddings
emotion2vec+ large
1-second windows, 0.5s stride. Produces 768-dim emotion embeddings + Praat features (F0, jitter, shimmer, HNR).
💾
Cache Embeddings
EmbeddingCache → MinIO
All extracted features cached for retraining. Landmark data, audio embeddings, and transcript stored with temporal alignment.
Analysis Engines
TRI-VECTOR ANALYSIS
👀
Gaze Deviation
TCN on landmarks → z-score
Score per statement window. Compare gaze patterns against contact baseline. Output: deviation_score, blink_anomaly, head_pose_delta per segment.
🔊
Tonality Flux
emotion2vec + Praat → XGBoost
Cosine distance between consecutive embedding windows. Praat delta from baseline. Output: flux_score, stress_indicators per segment.
📝
Contradiction NLP
DeBERTa-v3 + pgvector
Each statement embedded → top-5 similar history retrieved → NLI classification. Output: contradiction_score, matched_statements per segment.
🎯
Composite Scorer
XGBoost + Platt Calibration
12 features per window → 0-100 calibrated probability. Per-vector breakdown. SHAP explainability values. Confidence interval based on baseline quality.
📊
Report Generation
POST to PostgreSQL + push WS
Timestamped analysis report stored. Statement-level scores with transcript alignment. Push notification to client via WebSocket. Status → "complete".
Storage Lifecycle
🔥
Hot Tier (0-30d)
SSD — Full Payload
AV1 re-encoded video + audio + landmarks + transcript + scores. Instant playback. ~$1.15/user/mo.
30 DAYS
♨️
Warm Tier (30-90d)
HDD/S3-IA — Video Deleted
Audio + landmarks + transcript + scores. Video purged. Can re-derive gaze analysis from landmarks. ~$0.22/user/mo.
90 DAYS
❄️
Cold Tier (90-365d)
Glacier — Scores Only
Transcript + scores + statement embeddings. Enough for contradiction history. ~$0.04/user/mo.
365 DAYS
🗑️
Purge
GDPR / BIPA Compliant
All PII deleted. Only anonymized aggregate training data retained with explicit user consent.
Processing Time Estimates

A 20-minute uploaded video takes approximately 3-5 minutes to fully process (extraction + all three analysis vectors). The bottleneck is Whisper transcription and emotion2vec embedding extraction, which run in parallel. The client receives real-time progress updates: "Extracting audio…" → "Transcribing…" → "Analyzing gaze…" → "Scoring…" → "Complete."

Integration Strategy — Abstraction Layer

Each platform implements a PlatformConnector interface with four methods: authorize(), join_meeting(), stream_media(), and get_recording(). The connector abstracts whether media arrives as a real-time stream (Zoom SDK, LiveKit) or as a post-meeting recording download (Teams Graph API, Meet). The analysis pipeline consumes from either path identically.

Authorization & Consent
🔗
OAuth2 Connect
Platform-specific OAuth flow
User authorizes Veritas to access their platform account. Scopes: read meetings, join as bot/observer, access recordings. Refresh tokens stored in Vault.
📋
Meeting Detection
Webhooks / Calendar Sync
Platform webhook fires on meeting start (Zoom: meeting.started, Teams: /communications/callRecords). Or user manually triggers from app.
⚠️
Dual Consent Gate
Both participants must consent
In-app consent modal pushed to both users. If the other party doesn't have Veritas, they receive a web link to a consent form. Recording blocked until both consent tokens received. Tokens are signed JWTs stored for audit.
REQUIRED
✅
Analysis Enabled
Consent tokens verified
Both consent tokens validated. Platform connector activated. Media pipeline initiated. Either user can revoke at any time → immediate stop.
Media Acquisition (Two Paths)
PATH A — REAL-TIME STREAM
🎥
Bot Joins Meeting
Zoom SDK Bot / Teams Bot / LiveKit
Veritas bot joins as silent observer. Receives raw media streams (video + audio) from the platform's SFU. For Zoom: Meeting SDK media stream. For Teams: Bot media platform. For LiveKit: direct WebRTC subscription.
📡
Live Media Router
Kafka ingestion → frame dispatch
Audio chunks (200ms) → transcription pipeline. Video frames (5fps sample) → MediaPipe landmark extraction. Both feed into Kafka topics for parallel real-time analysis.
OR
PATH B — POST-CALL RECORDING
☁️
Recording Webhook
Platform posts recording URL
After meeting ends, platform fires webhook with recording download URL. Zoom: recording.completed. Teams: callRecording available. Google Meet: recording in Drive.
⬇️
Recording Download
Celery worker fetches file
Worker downloads recording from platform cloud. Stored in MinIO. Then follows the same extraction pipeline as manual file upload (FFmpeg demux → parallel extraction).
CONVERGE
🔀
Unified Pipeline
PlatformConnector abstraction
Both paths feed the same extraction + analysis pipeline. Real-time path produces incremental scores. Post-call path produces batch report. Same output schema.
Real-Time Analysis (Path A Only)
👁️
Live Gaze
MediaPipe → TCN (5fps sample)
Landmark extraction at 5fps (sufficient for gaze analysis, saves compute). TCN scores every 2-second window. Latency: ~500ms.
🎤
Live Transcription
Whisper distil-large-v3 (streaming)
Streaming VAD-triggered transcription. Statement boundary detection. Each completed statement pushed to contradiction pipeline. Latency: ~2s.
🔊
Live Tonality
Librosa handcrafted (fast path)
On-server handcrafted features (MFCC, F0, jitter) for real-time speed. Full emotion2vec analysis runs post-call for accuracy. Latency: ~300ms.
🎯
Live Score Push
XGBoost → WebSocket
Composite score computed every statement boundary. Pushed to client via WebSocket. Includes per-vector breakdown and confidence level. Total latency: ~2-3s from speech.
Post-Call Enhancement
🔄
Full Re-Analysis
emotion2vec + full Whisper
After call ends, run full accuracy pipeline (emotion2vec large, Whisper large-v3) on the complete recording. Produces higher-accuracy scores than real-time.
📈
Baseline Update
Per-contact stats refresh
Update contact's gaze baseline (mean/std), tonality baseline (F0 range, speaking rate), and linguistic profile (hedging frequency, vocabulary). Exponential moving average.
🧠
Adapter Retrain Check
Every 3rd call triggers retrain
If call_count % 3 == 0, queue XGBoost scorer retraining with all historical features for this contact. ~$0.15/retrain. Version new adapter in MinIO.
📊
Enhanced Report
Replaces live scores
Final report replaces real-time scores with full-accuracy scores. User notified "Enhanced analysis ready." Shows score delta from live vs. enhanced.
Platform Connector Details

Zoom: Meeting SDK (native) for real-time media access. Raw Media component captures audio/video streams directly. Requires Zoom Marketplace app approval.
Teams: Bot Framework + Communications API. Bot joins as participant via Graph API. Real-time media platform for audio/video. Requires Azure AD app registration + admin consent.
Google Meet: Meet REST API (GA 2024) + Meet Add-ons SDK for in-meeting integration. Cloud recording download via Drive API for post-call. Requires Google Workspace Marketplace listing.
Webex: Meetings API for scheduling + Webhooks for meeting events. Recording download via API. Bot capability for real-time access.
Slack: Events API for huddle detection. RTMP-based media relay for audio. Limited video access — audio-only analysis for Slack.
LiveKit (Native): Direct WebRTC with full media control. Lowest latency. Best quality. Used when both parties are on Veritas native calls.

⚠️
STEP 5a

Dual Consent Collection

Screen: Consent Modal (Both Parties)

Critical legal gate. Both participants must explicitly consent before any recording or analysis begins.

If other party has Veritas installed: In-app consent modal appears on their device: "This call will be recorded and analyzed by Veritas AI. Your behavioral data will be processed. Do you consent?"

If other party does NOT have Veritas: They receive an SMS/email with a web link to a consent form. The form explains what data is collected and how it's used. Consent is time-bound to this specific meeting.

I Consent Decline (call continues, no recording) Learn More (explainer page)

If either party declines: the call proceeds normally but with zero recording or analysis. The user is notified: "Analysis unavailable — consent not granted by all participants."

🎥
STEP 6a

Live Call with Real-Time Scoring

Screen: Active Call Overlay / Companion View

During the call, the user sees a subtle companion overlay (or a separate split-screen panel on tablet). The overlay shows:

Live Elements: Rolling composite score (0-100 gauge), per-vector mini indicators (eye icon for gaze, waveform for tonality, text icon for NLP), live transcript with highlighted flagged statements, and a "confidence level" badge based on how much baseline data exists for this contact.

UX Constraint: The overlay must NOT be visible to the other participant. On native calls (LiveKit), this is guaranteed. On platform calls (Zoom/Teams), the overlay runs in the Veritas app alongside the platform app — it's a separate window/notification layer.

Minimize Overlay Expand to Full Analysis Flag Moment (manual bookmark) Revoke Consent (stop analysis)
📊
STEP 7a

Post-Call Report

Screen: Analysis Report

After the call ends, the enhanced analysis runs (full-accuracy models). User receives a push notification when ready (3-5 minutes). The report contains:

Summary Card: Overall score, per-vector breakdown, confidence level, comparison to previous calls with this contact.

Timeline View: Scrubable timeline synced with transcript. Each statement has individual scores. Flagged moments highlighted. Tap any statement to see: gaze deviation chart at that moment, tonality waveform, and any contradictions found with historical statements (showing the matched statement and when it was said).

Trends View: If 3+ calls exist with this contact, shows score trends over time. "Baseline quality: Strong (5 calls)" or "Baseline quality: Low — 2 more calls needed for reliable scoring."
