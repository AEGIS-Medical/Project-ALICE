# ALICE — Protobuf Landmark Telemetry (Session 4)
## Design Spec · 2026-07-03

---

## Overview

Replace the streaming JSONL landmark artifact with a compact protobuf wire format,
closing gap #1 from CLAUDE.md's "Known gaps & next-session priorities": JSONL output
is ~12 MB/min against the edge-first telemetry budget, which makes EDGE_MINIMAL
uplinks (<1 Mbps) unable to carry landmark data at all. This gates the entire mobile
edge-first story.

**Decisions locked during brainstorming:**

1. **Scope: landmark telemetry only.** The `ScoreEvent` streaming schema (gap #2) is
   deferred to its own session — no consumer exists yet (YAGNI).
2. **Full 478-point mesh** is transmitted (not a landmark subset, not tiered cadence).
   Keeps every downstream option open: server AU detection, bilateral asymmetry,
   gaze, future re-ID alignment all want full geometry.
3. **Budget honestly restated.** The original ~70 KB/min figure is unreachable for a
   full raw mesh at 30 fps (even uint16-quantized, that is ~5.2 MB/min before deltas).
   The raw-landmark channel targets **≤ 1.2 MB/min at 30 fps** (bandwidth-derived test
   gate), measured at ~1.04 MB/min with 12-bit quantization — ~12x smaller than
   JSONL's ~12 MB/min. The 70 KB/min figure still belongs to the future on-device
   AU-activation payload (custom TFLite model, unbuilt), at which point raw landmarks
   become a fallback/debug channel. CLAUDE.md is updated to say this explicitly.
4. **JSONL is replaced outright** — no dual-write, no opt-in debug format. Nothing
   downstream consumes JSONL yet, so this is the cheapest moment to swap. A decoder
   CLI (`scripts/inspect_landmarks.py`) replaces JSONL's eyeball-ability.
5. **Approach: protobuf + keyframe/delta + zlib chunk framing** (Approach 1 of 3
   considered). Hand-rolled binary (Approach 2) rejected — owning a codec spec
   forever to save ~10-15% contradicts CLAUDE.md's protobuf direction. zstd +
   column-major packing (Approach 3) rejected for now — extra native dependency on
   mobile and it complicates crash recovery; it remains a future optimization inside
   the same container format.

**Cross-platform contract:** the `.proto` file is the source of truth, not the Python
code. Android (official Kotlin/Java protoc gen, or KMP via Wire/pbandk), iOS
(swift-protobuf or KMP-native bindings), and zlib is built into both OSes
(`java.util.zip` / `libz`). The chunk header carries a compression-method byte so an
uncompressed escape hatch exists if iOS cinterop zlib ever becomes a pain. The Kotlin
decoder is NOT built this session — it ships with the mobile client and generates
from the same `.proto`.

## Size Model (measured)

Per frame, full mesh: 478 landmarks × (x, y, z).

| Stage | Bytes/frame (approx) | MB or KB/min @30fps |
|---|---|---|
| Today: JSONL float text | ~12,000 | ~12 MB/min |
| uint12/int12 quantized, absolute | ~2,151 (12-bit packed as varints) | ~3.9 MB/min |
| Delta frames, zigzag varint (small motion → 1 byte/value) | ~1,000–1,400 | ~1.8–2.5 MB/min |
| + zlib over 30-frame chunks (correlated deltas) | measured 1,066,783 bytes / 60 s | **~1,042 KB/min ≈ 0.14 Mbps** |

Keyframes (every 30 frames) cost ~2.9 KB each ≈ 87 KB/min of the total — acceptable.
At EDGE_MINIMAL's 10 fps landmark cadence landmarks are ~350 KB/min ≈ 0.05 Mbps.

Real demo clip (trial_truth_001.mp4, EDGE_FULL): measured 1,196 KB/min — 0.3% under the gate, near the line by design; the operational safety margin lives at the uplink budget (combined stream <0.5 Mbps on a <1 Mbps EDGE_MINIMAL link), not the gate.

**Bandwidth-derived acceptance gate:** gate ≤ 1.2 MB/min @30 fps (≈ 0.16 Mbps); FLAC
0.33 Mbps + landmarks 0.16 Mbps < 0.5 Mbps on a <1 Mbps EDGE_MINIMAL uplink (2x
headroom); at EDGE_MINIMAL's 10 fps cadence landmarks are ~350 KB/min ≈ 0.05 Mbps.

## Wire Format

### `proto/landmarks.proto` (proto3)

```protobuf
syntax = "proto3";
package alice.telemetry.v1;

// CLAUDE.md invariant #9: this schema carries landmark geometry ONLY.
// Raw image/video data must NEVER be added to any message in this file.

message LandmarkStreamHeader {
  uint32 version = 1;              // format version, starts at 1
  uint32 landmark_count = 2;       // 478
  float  source_fps = 3;
  uint32 keyframe_interval = 4;    // frames between keyframes (default 30)
  uint32 frame_skip = 5;           // extractor's frame_skip setting
}

message KeyFrame {
  uint32 frame_number = 1;
  repeated uint32 xy = 2 [packed = true];  // 956 values: x0,y0,x1,y1... uint12-quantized (scale 4095)
  repeated sint32 z  = 3 [packed = true];  // 478 values: int12-quantized (scale 2047, zigzag)
}

message DeltaFrame {
  uint32 frame_number = 1;
  repeated sint32 dxy = 2 [packed = true]; // deltas vs previous frame, zigzag varint
  repeated sint32 dz  = 3 [packed = true];
}

message NoFaceFrame {                       // preserves JSONL's "landmarks: null" rows
  uint32 frame_number = 1;
}

message Frame {
  oneof kind {
    KeyFrame key = 1;
    DeltaFrame delta = 2;
    NoFaceFrame no_face = 3;
  }
}

message LandmarkChunk {
  repeated Frame frames = 1;                // up to keyframe_interval frames
}
```

### Quantization

- x, y are MediaPipe-normalized `[0, 1]` → `round(v * 4095)` as uint12.
  Max reconstruction error `1/8190` ≈ **0.13 px at 1080p** — below MediaPipe's own
  detector jitter (~0.2 px), so quantization is invisible downstream. uint16 and an
  initial 500 KB gate were revised after measurement showed sub-jitter precision
  encodes incompressible detector noise (uint16 measured 5.3x over the bandwidth
  envelope). Out-of-range inputs are clamped to `[0, 1]` and counted in encoder
  telemetry (`clamped_values`).
- z is approximately `[-1, 1]` → `round(v * 2047)` clamped to int12.
- Timestamps are NOT stored per frame: `timestamp_seconds = frame_number / source_fps`,
  reconstructed exactly as the extractor computes them today (header carries fps).

### Delta rule

- A frame is a `DeltaFrame` iff the previous *emitted* frame contained a face AND the
  frame is not on a keyframe slot.
- After any `NoFaceFrame`, the next face frame is forced to be a `KeyFrame` (no stale
  delta reference).
- A `KeyFrame` is forced every `keyframe_interval` (default 30) emitted frames.

### File framing

```
MAGIC "ALTM" (4 bytes)
u32 header_length | serialized LandmarkStreamHeader
repeat:
  u32 chunk_length | u8 compression_method | zlib(serialized LandmarkChunk)
```

- `compression_method`: 1 = zlib (default), 0 = uncompressed (escape hatch).
- All integer prefixes are little-endian.
- `chunk_size == keyframe_interval`, so **every chunk starts at a keyframe** — any
  file prefix ending on a chunk boundary is fully decodable. This preserves P1-S6's
  crash-recovery guarantee (a killed process loses at most the tail chunk).

## Pipeline Integration

`FeatureExtractor.extract_landmarks()` keeps its exact signature and `Path` return.
The JSONL writer block becomes:

```python
output_path = output_dir / f"{video_path.stem}_landmarks.pb"
with LandmarkEncoder(output_path, source_fps=fps,
                     frame_skip=self.frame_skip) as encoder:
    while ...:                                  # existing MediaPipe loop, unchanged
        encoder.add_frame(frame_idx, landmarks)  # landmarks list or None
```

- `flush_interval` (P1-S6) is superseded by `keyframe_interval` (same default, 30 =
  one chunk per second at 30 fps). The parameter remains accepted and maps onto
  `keyframe_interval`, with a deprecation note in the docstring, so existing call
  sites do not break.
- `pipeline.py`: **no code change** (it consumes only `landmarks_path` + file size).
- `CompressionResult.landmarks_path` docstring updated to `.pb`.
- `scripts/test_compress_and_analyze.py`: print label "Landmarks (JSONL)" →
  "Landmarks (protobuf)".
- Telemetry attributes (`last_frames_processed`, `last_frames_with_face`) unchanged.

### `scripts/inspect_landmarks.py`

The JSONL-replacement debugging tool. Prints: header fields, frame/chunk counts,
face-coverage %, bytes/min at source fps, max quantization error on a sampled frame,
and the JSONL-equivalent size for contrast. `--head N` dumps the first N decoded
frames as JSON.

## Error Handling

### Encoder

| Failure | Handling |
|---|---|
| Wrong landmark count in `add_frame` | `ValueError` immediately — malformed frames never reach the wire |
| Coordinate outside [0,1] (or z outside [-1,1]) | Clamp before quantizing; counted in `clamped_values` telemetry |
| Disk-full / IO error mid-write | Propagates (same policy as JSONL); file remains decodable to last complete chunk |
| Process killed mid-call | No corruption by construction: header at open, atomic length-prefixed chunks; torn tail chunk detected by length mismatch and skipped on decode |
| `close()` twice / zero frames | Idempotent; empty file = magic + header + no chunks → decodes to zero frames |

### Decoder

| Failure | Handling |
|---|---|
| Bad magic | `ValueError("not an ALICE landmark telemetry file")` at construction |
| Unsupported header version | `ValueError` naming the version (forward-compat gate) |
| Truncated final chunk | Warn + stop iteration — expected post-crash, not an error |
| Corrupt zlib inside a length-valid chunk | `LandmarkDecodeError` naming the chunk index — real damage must not be silent |
| DeltaFrame with no prior keyframe | `LandmarkDecodeError` — encoder cannot produce this; indicates tampering/bugs |

### CLAUDE.md invariants

- **#9 (no raw video in edge-first):** schema carries landmarks only; the `.proto`
  header comment forbids adding image data.
- **#3 (no PII in logs):** codec logs counts/bytes/ratios only — never coordinates.
- **#1 (lossless audio):** untouched. Landmark quantization is lossy by design; the
  documented error bound (≤ 0.13 px @1080p) is below MediaPipe's own detector jitter (~0.2 px).

## Testing

### Unit — `tests/telemetry/`

| File | Coverage |
|---|---|
| `test_quantization.py` | Round-trip error ≤ 1/4094 for random [0,1] floats; clamping of out-of-range; z signed range |

### The budget gate — the point of the session

`test_codec_roundtrip.py::test_bytes_per_minute_budget`: a synthetic 60 s @ 30 fps
stream with realistic motion (small Gaussian per-frame deltas + occasional saccade
jumps + brief no-face gap) must encode to **≤ 1.2 MB** (bandwidth-derived gate).
Hard failure if over.

### Integration — `tests/compression/test_feature_extractor.py` (updated)

- `extract_landmarks` on a real demo video → `.pb` output, decoder round-trips it,
